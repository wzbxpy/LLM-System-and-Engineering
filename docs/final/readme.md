# 期末报告说明 (Final Project Report)

## 背景
本学期课程中，部分同学在助教指导下参与了研究项目。**只有参加项目的同学需要提交期末报告**，以项目组为单位进行提交。

## 提交要求

### 1. 提交形式
以**项目组**为单位提交一份期末报告（PDF格式），报告需以**学术论文**的形式组织。

### 2. 报告类型
你们可以选择以下任一类型：
- **Survey（综述论文）**：对项目相关领域的研究现状进行系统性梳理与总结，包括关键技术、代表性工作、发展趋势等。
- **Research Paper（研究论文）**：呈现项目的研究问题、方法设计、实验结果与分析。鼓励提出创新性的想法或初步的实验验证。

### 3. 格式要求
请参考以下学术会议的论文模板（任选其一）：
- **ACM格式**：适用于系统、架构、数据库等方向（如 SOSP, OSDI, SIGMOD）
- **IEEE格式**：适用于网络、安全等方向
- **USENIX格式**：适用于系统与安全方向

**页数要求**：
- 建议 6-12 页（不含参考文献）

**内容要求**：
- 标题、作者信息（项目组成员 + 指导助教）
- 摘要 (Abstract)
- 引言 (Introduction)
- 相关工作 (Related Work)
- 方法设计/技术综述 (Methodology/Survey)
- 实验/案例分析 (Evaluation，如适用)
- 结论与未来工作 (Conclusion and Future Work)
- 参考文献 (References)

### 4. 评分标准
- **内容完整性**（40%）：是否清晰地阐述了研究问题、方法或综述内容？
- **技术深度**（30%）：是否展现了对问题的深入理解和思考？
- **创新性/洞察力**（20%）：是否提出了新的想法或有价值的总结？
- **写作质量**（10%）：逻辑是否清晰，表达是否规范？
