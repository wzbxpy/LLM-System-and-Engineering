# Assignment 2: 审稿意见评审 (Meta-Review)

[Assignment 1 审稿意见](https://table.nju.edu.cn/dtable/links/7cf08b8d3ba04cb5b6f6)已全部收集完成。在 Assignment 2 中，每位同学将沿用 Assignment 1 时的分组，作为“Meta-reviewer”为其他分组同学提交的审稿意见质量进行评分。

## 目标
本次作业旨在培养大家鉴别“高质量审稿意见”的能力。一份好的审稿意见不仅能决定论文的去留，更应该能帮助作者改进工作。

## 具体要求

### 1. 评分策略与分布控制
*   **模拟真实会议接受率**：当前系统方向顶级会议（如 SOSP, OSDI, EuroSys 等）的接受率通常在 **15%-18%** 左右。
*   **硬性约束**：每个组在评审分配到的 8 篇论文时，给出的 **Accept (4分)** 及 **Strong Accept (5分)** 的总数**不得超过 3 篇**。请在组内讨论并达成共识，筛选出真正值得发表的工作。

### 2. 评价审稿意见的质量 (Meta-Review Criteria)
请从以下维度对 Assignment 1 的审稿意见进行打分（1-5分）和点评：

*   **详实程度 (Informativeness)**：审稿意见是否提供了足够的细节？Detailed Feedback 是否具体指出了问题所在，还是仅有空泛的评论？
*   **论据支撑 (Justification)**：给出的分数（Overall Merit）是否与 Strength 和 Weakness 中的描述一致？拒绝或接受的理由是否充分？
*   **建设性 (Constructiveness)**：是否给出了具体的改进建议？语气是否专业且尊重作者？
*   **准确性 (Correctness)**：审稿人是否正确理解了论文的核心贡献？是否存在明显的误读？

### 3. 提交格式与内容
以**组**为粒度提交一份报告（PDF或Markdown格式），包含以下内容：

1.  **最终决策列表**：列出你们组负责评审的 8 篇论文的最终推荐结果（Accept/Reject）及排序。
2.  **各组审稿质量评分**：
    *   对其他组提交的评审意见进行打分（百分制）。
    *   **评分依据**：针对给出的分数，请提供简要的理由或评语（Meta-Review Comment）。可以具体指出该组在哪些论文的评审中表现出色或不足。
    *   格式示例：**1组：80分。1组在Paper 3的评审中非常详尽，但在Paper 5中遗漏了关键问题...**

### 4. 评分标准 (Rubric)
*   **100分 (Excellent)**：审稿意见极其详尽，指出了论文的关键痛点或亮点，对作者改进论文有巨大帮助。
*   **80分 (Average)**：完成了基本的审稿要求，指出了主要优缺点，但缺乏深度或具体建议。
*   **60分 (Poor)**：审稿意见过短（如仅有一两句话），甚至存在明显的理解错误，或语气不当。
